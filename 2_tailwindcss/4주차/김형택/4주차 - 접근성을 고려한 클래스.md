# ♿️ 접근성을 고려한 Tailwind CSS 사용법 요약


## 1. 스크린 리더 전용 클래스: `sr-only`

`sr-only` 클래스는 **시각적으로는 숨기지만 스크린 리더에게는 읽히도록**
만드는 Tailwind의 접근성 핵심 클래스입니다.

### ✔ 역할

-   **시각적 숨김:** 요소가 화면에 보이지 않도록 1px 영역으로 클리핑
-   **스크린 리더 노출:** DOM에는 그대로 존재 → 보조기기는 정상 인식

### ✔ 사용 예시

#### 아이콘 버튼에 숨겨진 텍스트 레이블 추가

``` html
<button class="bg-blue-500 hover:bg-blue-700 text-white font-bold p-3 rounded">
  <svg class="w-5 h-5" fill="currentColor" viewBox="0 0 20 20">
    <path d="M10 3a1 1 0 011 1v2a1 1 0 11-2 0V4a1 1 0 011-1z" />
  </svg>

  <span class="sr-only">검색 필드 초기화</span>
</button>
```

#### 시각적으로 숨겨진 레이블 제공

``` html
<label for="search">
  <span class="sr-only">검색어 입력</span>
  <input id="search" type="text" placeholder="검색" class="p-2 border rounded">
</label>
```

## 2. 포커스 관리 Variant: `focus-visible`


`focus-visible`은 **사용 방식(키보드/마우스)에 따라 포커스 표시를 더 적절하게 제어하는 기능**입니다.  
`focus:outline-none`으로 기본 아웃라인을 제거하고, 대신 `focus-visible`을 사용해 **키보드 사용 시에만 명확한 포커스 강조 스타일을 적용**할 수 있습니다.  

이를 통해 불필요한 포커스 표시(마우스 클릭)를 숨기면서도,  
**키보드 탐색 사용자에게는 시각적으로 더 잘 보이는 포커스를 제공**할 수 있습니다.

> **즉, 마우스 사용자에게는 포커스를 숨기고,  
> 키보드 사용자에게는 더 효과적인 시각적 강조를 제공하는 것이 이 조합의 목적이다.**

### ✔ 해결하는 문제

-   마우스 클릭 → 포커스 시 스타일 불필요
-   키보드(Tab) 이동 → 반드시 포커스 표시 필요\
    → `focus-visible`이 이를 자동으로 구분

### ✔ 사용 예시

#### 키보드 사용자에게만 링 효과 제공

``` html
<button class="p-2 border rounded 
               focus:outline-none 
               focus-visible:ring-4 
               focus-visible:ring-offset-2 
               focus-visible:ring-indigo-500">
  키보드 전용 포커스 스타일
</button>
```

#### 링크 포커스 강조

``` html
<a href="#" class="inline-block p-2 text-indigo-600 
                   focus:outline-none 
                   focus-visible:underline 
                   focus-visible:font-bold">
  키보드로 링크 이동 시 강조
</a>
```
